# pages-mksearch

## 简介

`pages-mksearch` 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅用于存放和托管各种静态 HTML 文件，方便通过 GitHub Pages 或其他静态托管服务进行访问与分享。

## 目录说明

```
.
├── index.html          # 仓库主页入口
├── pages/              # 存放各个独立 HTML 页面
│   ├── example1.html
│   ├── example2.html
│   └── ...
├── assets/             # 静态资源文件（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- **index.html**：仓库的默认入口页面，通常包含页面列表或导航信息。
- **pages/**：所有独立 HTML 页面存放于此目录，每个文件对应一个独立页面。
- **assets/**：用于存放页面所需的样式、脚本和图片等资源文件。

## 页面归档说明

本仓库用于归档那些不依赖特定后端服务、可直接在浏览器中运行的 HTML 页面。每个页面应保持自包含性，尽量将样式和脚本内联或引用本仓库内的资源文件。

### 归档原则

- 页面应独立完整，避免跨域引用外部资源（除非必要）。
- 文件命名应清晰反映页面内容或用途。
- 如需引用图片或样式，请优先使用 `assets/` 目录下的本地资源。

## 维护说明

- **添加新页面**：在 `pages/` 目录下创建新的 HTML 文件，并确保其引用资源路径正确。
- **更新页面**：直接修改对应文件，建议保留版本历史记录。
- **删除页面**：移除对应文件，并更新 `index.html` 中的导航链接（如有）。
- **资源管理**：请及时清理未使用的资源文件，保持仓库整洁。

## 使用方式

1. 克隆本仓库到本地。
2. 在 `pages/` 目录下添加或修改 HTML 文件。
3. 提交并推送更改。
4. 启用 GitHub Pages 后，即可通过仓库的 Pages 链接访问各页面。
